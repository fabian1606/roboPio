Diese Anleitung führt durch die komplette Einrichtung des roboPio-Projekts – von der Erstinstallation bis zum ersten Test.

## Voraussetzungen

- **Arduino IDE** (Version 2.x empfohlen) oder **PlatformIO** (für den Empfänger)
- **ESP32-Arduino-Kern** installiert (Board-Manager-URL: `https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json`)
- USB-Kabel für ESP32-S3 (USB-C oder Micro-USB je nach Board)

---

## Einrichtung

<Steps>

1. ### RobotLink-Bibliothek herunterladen

   Die neueste Version der Bibliothek als ZIP herunterladen:

   **[→ Zum neuesten Release auf GitHub](https://github.com/fabian1606/roboPio/releases/latest)**

   Dort die Datei `RobotLink-x.y.z.zip` herunterladen.

2. ### RobotLink-Bibliothek installieren

   **In der Arduino IDE:**

   1. Menü: **Sketch → Bibliothek einbinden → .ZIP-Bibliothek hinzufügen…**
   2. Die heruntergeladene `RobotLink-x.y.z.zip` auswählen.
   3. Die Arduino IDE bestätigt die Installation mit einer Erfolgsmeldung.

   <Aside type="tip">
     Die Bibliothek erscheint danach unter **Sketch → Bibliothek einbinden → RobotLink** und kann mit `#include <RobotLink.h>` eingebunden werden.
   </Aside>

3. ### Abhängigkeiten installieren

   Folgende Bibliotheken müssen zusätzlich im **Library Manager** installiert werden (**Sketch → Bibliothek einbinden → Bibliotheken verwalten…**):

   | Bibliothek | Version | Beschreibung |
   |------------|---------|--------------|
   | **FastLED** | ~3.9.0 | RGB-LED-Steuerung (WS2812B) |
   | **Smoothed** | ^1.2 | Glättungsfilter für analoge Eingaben |

   Einfach den Bibliotheksnamen in die Suchleiste eingeben und auf **Installieren** klicken.

   <Aside type="note">
     Beim ZIP-Import bietet die Arduino IDE evtl. auch **SCServo** als Abhängigkeit an. Diese Bibliothek wird nur vom **Empfänger** (PlatformIO) benötigt – für den Sender ist sie nicht erforderlich, schadet aber auch nicht.
   </Aside>

4. ### Board-Einstellungen konfigurieren (Sender)

   Für den **Sender (ESP32-S3)** folgende Board-Einstellungen in der Arduino IDE wählen:

   | Einstellung | Wert |
   |------------|------|
   | Board | `ESP32S3 Dev Module` |
   | USB Mode | `Hardware CDC and JTAG` |
   | Upload Speed | `921600` |
   | Flash Size | `4MB (32Mb)` |

   Das richtige Board auswählen: **Werkzeuge → Board → esp32 → ESP32S3 Dev Module**.

5. ### Beispielprojekt öffnen

   Die RobotLink-Bibliothek enthält fertige Beispiele für jeden Koordinatenmodus:

   **Arduino IDE:** Datei → Beispiele → RobotLink → gewünschtes Beispiel

   | Beispiel | Koordinatenmodus |
   |----------|-----------------|
   | `SenderDirect` | Direkt (1:1 Poti → Servo) |
   | `SenderCylindrical` | Zylindrisch (Basiswinkel, Elevation, Reichweite) |
   | `SenderCartesian` | Kartesisch (X, Y, Z) |

   <Aside>
     Der **Gruppenkanal** (LED-Farbe, 1–5) ist unabhängig vom Koordinatenmodus. Sender und Empfänger müssen auf demselben Kanal eingestellt sein. Der Kanal wird per Doppel-Reset gewechselt und bleibt im EEPROM gespeichert.
   </Aside>

6. ### Empfänger einrichten (PlatformIO)

   Der Empfänger wird als **PlatformIO**-Projekt im roboPio-Repository bereitgestellt:

   ```bash
   git clone https://github.com/fabian1606/roboPio
   cd roboPio
   pio run -e receiver -t upload
   ```

   Alternativ in VS Code: PlatformIO-Erweiterung installieren, Ordner `roboPio/` öffnen, dann **PlatformIO: Upload**.

7. ### Sender und Empfänger auf denselben Gruppenkanal stellen

   Eine **MAC-Adresse muss nicht eingetragen werden.** Der Sender überträgt
   per Broadcast, und der Empfänger akzeptiert ein Frame nur dann, wenn
   Sender und Empfänger auf demselben **Gruppenkanal (1–5)** stehen –
   erkennbar an der **gleichen LED-Farbe**.

   | Kanal | LED-Farbe |
   |-------|-----------|
   | 1 | Weiß |
   | 2 | Rot |
   | 3 | Grün |
   | 4 | Blau |
   | 5 | Magenta |

   So gleichst du den Kanal ab:

   1. LED-Farbe von Sender und Empfänger vergleichen.
   2. Stimmen sie nicht überein, auf einem der Boards den Kanal per
      **Doppel-Reset** (zwei RST-Drücke innerhalb von 400 ms) weiterschalten,
      bis beide dieselbe Farbe zeigen.

   <Aside type="tip">
     Dieses System ermöglicht, dass **mehrere Gruppen gleichzeitig** im selben
     Raum arbeiten: Jedes Sender-Empfänger-Paar nutzt einen eigenen Kanal und
     ignoriert die Frames der anderen Gruppen.
   </Aside>

8. ### Ersten Test durchführen

   1. Empfänger einschalten (Servos anschließen, 5V-Versorgung sicherstellen).
   2. Sender (ESP32-S3) per USB mit Strom versorgen und den Sketch hochladen.
   3. Die LED am Sender leuchtet in der Farbe des aktiven Gruppenkanals (weiß = Kanal 1).
   4. Potentiometer drehen → Servos bewegen sich.

   Bei Problemen: Seriellen Monitor (115200 Baud) öffnen – RobotLink gibt Statusinformationen aus.

</Steps>

## Verdrahtung des Senders

Die sechs Potentiometer werden an die ADC-fähigen GPIO-Pins des ESP32-S3 angeschlossen:

- **VCC** (äußerer Pin) → **3,3 V** des ESP32-S3
- **GND** (anderer äußerer Pin) → **GND** des ESP32-S3
- **Schleifer** (mittlerer Pin) → jeweiliger **GPIO-Pin** (siehe Modus-Dokumentation)

<Aside type="tip">
  Für den Direkt-Modus werden die GPIOs **17, 5, 6, 7, 15, 16** verwendet.  
  Für Zylindrisch- und Kartesisch-Modus: **4, 7, 6, 5, 15, 16**.
</Aside>
