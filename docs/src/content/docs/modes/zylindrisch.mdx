## Übersicht

Im **Zylindrisch-Modus** (`COORD_CYLINDRICAL`) wird der Roboterarm in **zylindrischen Koordinaten** gesteuert. Anstatt einzelne Gelenke direkt anzusteuern, gibt der Bediener folgende Parameter vor:

- **Basiswinkel ψ** – Rotation der gesamten Armstruktur um die vertikale Achse
- **Elevation γ** – Neigungswinkel des Arms nach oben/unten
- **Reichweite l** – horizontale Streckung des Arms (Abstand Basis–Werkzeug)
- **Handgelenk-Rotation θ** – Drehung des Werkzeugs um seine eigene Achse

Der Empfänger berechnet automatisch die benötigten Gelenkwinkel für Schulter und Ellbogen mittels eines **2-Gelenk-IK-Lösers**.

---

## Achsenbelegung

| Achse | GPIO | Funktion | Wertebereich |
|-------|------|----------|--------------|
| 0 | 4 | Basis-Rotation ψ | 0° – 360° |
| 1 | 7 | Elevation γ | 0° – 80° |
| 2 | 6 | Reichweite l | 40 – 240 mm |
| 3 | 5 | Handgelenk-Rotation θ | 0° – 360° |
| 4 | 15 | — (nicht verwendet) | — |
| 5 | 16 | Greifer | 0 – 100 % |

---

## Arm-Geometrie

Der 2-Gelenk-IK-Löser des Empfängers verwendet folgende Längenmaße:

| Segment | Länge | Beschreibung |
|---------|-------|--------------|
| L1 | 115 mm | Oberarm (Schulter → Ellbogen) |
| L2 | 135 mm | Unterarm (Ellbogen → Handgelenk) |
| L3 | 165 mm | Handgelenksverlängerung (Handgelenk → Werkzeugspitze) |

Der maximale Arm-Radius beträgt L1 + L2 + L3 = **415 mm**. Die praktisch sinnvolle Reichweite ist auf 240 mm begrenzt, um Singularitäten zu vermeiden.

---

## Handgelenk-Modi

Der Empfänger unterstützt zwei Handgelenk-Verhaltensweisen, die in der Receiver-Firmware konfiguriert werden:

| Modus | Konstante | Beschreibung |
|-------|-----------|--------------|
| Parallel zur Elevation | `WRIST_PARALLEL_GAMMA` | Das Handgelenk folgt dem Elevationswinkel – die Werkzeugachse bleibt parallel zur Armlinie. |
| Waagerecht | `WRIST_HORIZONTAL` | Das Handgelenk kompensiert die Elevation und bleibt immer horizontal (z. B. für Greifaufgaben auf einer Arbeitsfläche). |

## Beispielcode

```cpp
#include <RobotLink.h>

const int basePot    = 4;   // Achse 0 → Basis-Rotation ψ
const int elevPot    = 7;   // Achse 1 → Elevation γ
const int radiusPot  = 6;   // Achse 2 → Reichweite l
const int wristPot   = 5;   // Achse 3 → Handgelenk-Rotation θ
                            // Achse 4 → nicht verwendet
const int gripperPot = 16;  // Achse 5 → Greifer

const int smoothing = 10;   // Glättungsstärke (0 = aus, 50 = stark)

void setup() {
  robotLink.setAxisSmoothing(0, smoothing);
  robotLink.setAxisSmoothing(1, smoothing);
  robotLink.setAxisSmoothing(2, smoothing);
  robotLink.setAxisSmoothing(3, smoothing);
  // Achse 4 unbenutzt – kein Smoothing nötig
  robotLink.setAxisSmoothing(5, smoothing);

  robotLink.setCoordMode(COORD_CYLINDRICAL);
  robotLink.beginSender();
}

void loop() {
  robotLink.update();

  robotLink.setAxisValue(0, analogRead(basePot));
  robotLink.setAxisValue(1, analogRead(elevPot));
  robotLink.setAxisValue(2, analogRead(radiusPot));
  robotLink.setAxisValue(3, analogRead(wristPot));
  robotLink.setAxisValue(4, 0);  // unbenutzt
  robotLink.setAxisValue(5, analogRead(gripperPot));

  robotLink.sendAllAxes();
  delay(20);  // 50 Hz
}
```

<Aside>
  Der Gruppenkanal (LED-Farbe) ist unabhängig vom Koordinatenmodus. Sender und Empfänger müssen auf demselben Kanal sein – der Kanal wird per Doppel-Reset gewechselt.
</Aside>

## Funktionsweise der inversen Kinematik

### Koordinatentransformation

Der Empfänger wandelt die zylindrischen Koordinaten (ψ, γ, l) wie folgt in kartesische Koordinaten um:

```
x = l · cos(ψ)
y = l · sin(ψ)
z = l · tan(γ)
```

### 2-Gelenk-IK-Löser

Aus den kartesischen Koordinaten wird der Abstand `d` zur Armwurzel berechnet. Anschließend löst der Cosinus-Satz die Gelenkwinkel:

```
cos(θ_Ellbogen) = (d² - L1² - L2²) / (2 · L1 · L2)
θ_Schulter     = atan2(z, r) - atan2(L2 · sin(θ_Ellbogen), L1 + L2 · cos(θ_Ellbogen))
```

<Aside type="caution">
  Liegt der Zielpunkt außerhalb des Arbeitsraums (l > L1 + L2), streckt der Empfänger den Arm maximal aus, ohne einen Fehler zu erzeugen. Der Arm erreicht den Zielpunkt dann nicht exakt.
</Aside>
