## Übersicht

Im **Direkt-Modus** (`COORD_DIRECT`) wird jeder Servo direkt von einem Potentiometer gesteuert – ohne Koordinatentransformation oder inverse Kinematik. Der analoge Rohwert des jeweiligen ADC-Pins (0–4095) wird 1:1 auf den Servo-Winkel abgebildet.

Dieser Modus eignet sich besonders gut für:

- **Erstinbetriebnahme** und Kalibrierung einzelner Gelenke
- **Diagnose** bei Fehlfunktionen (jede Achse unabhängig prüfbar)
- **Einfache Bewegungsabläufe**, bei denen absolute Koordinaten nicht benötigt werden

## Achsenbelegung

| Achse | GPIO | Funktion | Servo |
|-------|------|----------|-------|
| 0 | 17 | Basis-Rotation | 1 |
| 1 | 5 | Schulter | 2 |
| 2 | 6 | Ellbogen | 3 |
| 3 | 7 | Handgelenk-Neigung | 4 |
| 4 | 15 | Handgelenk-Rotation | 5 |
| 5 | 16 | Greifer | 6 |

Jedes Potentiometer wird wie folgt angeschlossen:

| Potentiometer-Pin | Verbindung |
|-------------------|-----------|
| VCC (äußerer Pin) | 3,3 V |
| GND (anderer äußerer Pin) | GND |
| Schleifer (Mitte) | GPIO-Pin (s. Tabelle oben) |

<Aside type="tip">
  Der ESP32-S3 hat 12-Bit-ADCs (0–4095). Die RobotLink-Bibliothek übernimmt die Skalierung auf den Servo-Winkelbereich.
</Aside>

## Beispielcode

```cpp
#include <RobotLink.h>

// GPIO-Pin pro Servo
const int servo1Pin = 17;  // Basis-Rotation
const int servo2Pin = 5;   // Schulter
const int servo3Pin = 6;   // Ellbogen
const int servo4Pin = 7;   // Handgelenk-Neigung
const int servo5Pin = 15;  // Handgelenk-Rotation
const int servo6Pin = 16;  // Greifer

const int smoothing = 10;  // Glättungsstärke (0 = aus, 50 = stark)

void setup() {
  robotLink.setAxisSmoothing(0, smoothing);
  robotLink.setAxisSmoothing(1, smoothing);
  robotLink.setAxisSmoothing(2, smoothing);
  robotLink.setAxisSmoothing(3, smoothing);
  robotLink.setAxisSmoothing(4, smoothing);
  robotLink.setAxisSmoothing(5, smoothing);

  // COORD_DIRECT ist der Standard – kein setCoordMode() nötig
  robotLink.beginSender();
}

void loop() {
  robotLink.update();

  robotLink.setAxisValue(0, analogRead(servo1Pin));
  robotLink.setAxisValue(1, analogRead(servo2Pin));
  robotLink.setAxisValue(2, analogRead(servo3Pin));
  robotLink.setAxisValue(3, analogRead(servo4Pin));
  robotLink.setAxisValue(4, analogRead(servo5Pin));
  robotLink.setAxisValue(5, analogRead(servo6Pin));

  robotLink.sendAllAxes();
  delay(20);  // 50 Hz
}
```

<Aside>
  Der Gruppenkanal (LED-Farbe) ist unabhängig vom Koordinatenmodus. Sender und Empfänger müssen auf demselben Kanal sein – der Kanal wird per Doppel-Reset gewechselt.
</Aside>

## Funktionsweise im Detail

### Glättungsfilter

Der Glättungsparameter wendet eine **exponentielle Glättung** auf die ADC-Werte an (intern `SMOOTHED_EXPONENTIAL`). Höhere Werte glätten stärker: `1` bedeutet kaum Glättung, `50` sehr starke Glättung. Dies reduziert ADC-Rauschen und verhindert zitternde Servos.

```cpp
// Stärkere Glättung für die Basis (größere Trägheit gewünscht)
robotLink.setAxisSmoothing(0, 20);

// Keine Glättung für den Greifer (sofortige Reaktion)
robotLink.setAxisSmoothing(5, 0);
```

### Übertragungsrate

Die `delay(20)`-Anweisung ergibt eine Übertragungsrate von ca. **50 Hz**. Diese kann erhöht werden, führt aber zu mehr ESP-NOW-Paketen. Der Empfänger verarbeitet jeden empfangenen Wert sofort.
