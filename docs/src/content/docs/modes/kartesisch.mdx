## Übersicht

Im **Kartesisch-Modus** (`COORD_CARTESIAN`) wird die Position der **Werkzeugspitze** direkt im dreidimensionalen Raum vorgegeben. Die drei Potentiometer für X, Y und Z verschieben den Werkzeugmittelpunkt entlang der entsprechenden Achse; zwei weitere Regler steuern die Werkzeugausrichtung und die Greiferzange.

Der Empfänger berechnet in Echtzeit alle sechs Gelenkwinkel mithilfe eines **vollständigen 3D-Inverse-Kinematik-Lösers**.

Dieser Modus eignet sich für:

- **Präzisionsaufgaben**, bei denen der Endpunkt wichtiger ist als der Weg
- **Zusammenarbeit mit CNC- oder CAM-Planung** (Werkzeugpfade in kartesischen Koordinaten)
- **Pädagogische Zwecke** – intuitives Verstehen von Roboter-Arbeitsräumen

## Achsenbelegung

| Achse | GPIO | Funktion | Wertebereich |
|-------|------|----------|--------------|
| 0 | 4 | X-Position (vor/zurück) | 50 – 350 mm |
| 1 | 7 | Y-Position (links/rechts) | -200 – +200 mm |
| 2 | 6 | Z-Position (oben/unten) | -100 – +300 mm |
| 3 | 5 | Werkzeug-Neigung η | -90° – +90° |
| 4 | 15 | Handgelenk-Rotation | 0° – 360° |
| 5 | 16 | Greifer | 0 – 100 % |

## Koordinatensystem

Das Koordinatensystem ist wie folgt definiert:

- **Ursprung** – Mittelpunkt der Basisdrehachse auf Tischhöhe
- **X-Achse** – Vorwärtsrichtung des Arms in seiner Grundstellung
- **Y-Achse** – seitliche Richtung (links = negativ, rechts = positiv)
- **Z-Achse** – vertikale Richtung (oben = positiv)

### Ausrichtung drehen (Yaw-Offset)

Das gesamte Koordinatensystem lässt sich **zur Laufzeit um die Z-Achse drehen** – damit
verschiebt sich die Ausrichtung der X-/Y-Achse, ohne neu zu flashen. Mathematisch wird
lediglich ein Offset zum Basiswinkel addiert:

```
ψ = atan2(Y, X) + yawOffset
```

Der horizontale Radius `r = sqrt(X² + Y²)` bleibt dabei unverändert – eine Drehung um Z
verändert nur die Blickrichtung des Frames.

Der Wert wird als **Laufzeit-Parameter** `PARAM_YAW_OFFSET` über einen separaten
Funk-Kanal vom Sender zum Empfänger geschickt (nur bei Änderung) und dort **dauerhaft im
Flash gespeichert** (übersteht Resets). Die sechs Achs-Frames bleiben davon unberührt.

```cpp
// Sender: Yaw-Winkel (rad) setzen – z. B. aus einem zusätzlichen Poti
robotLink.sendParam(PARAM_YAW_OFFSET, yaw);

// Empfänger: aktuellen Wert auslesen
float yaw = robotLink.getParam(PARAM_YAW_OFFSET);
```

<Aside>
  Der Parameter-Kanal (`RobotParamFrame`) ist generisch: Weitere Laufzeit-Parameter lassen
  sich durch Ergänzen des `RobotParam`-Enums hinzufügen, ohne das Achs-Protokoll zu ändern.
</Aside>

### Maximalgeschwindigkeit

Über denselben Parameter-Kanal lässt sich die **maximale Servo-Geschwindigkeit** setzen
(gilt modusübergreifend für alle Servos). Die Einheit sind SMS-STS-Geschwindigkeitsschritte;
**`0` = unbegrenzt** (maximale Geschwindigkeit, Standard). Kleinere Werte ergeben sanftere,
langsamere Bewegungen. Der Wert wird am Empfänger dauerhaft gespeichert.

```cpp
robotLink.setMaxSpeed(0);      // 0 = max (Standard)
robotLink.setMaxSpeed(1500);   // langsamer / sanfter
```

## Verhalten an den Grenzen

Wenn eine angeforderte Position **außerhalb des erreichbaren Arbeitsraums oder der
Gelenk-Limits** liegt, bestimmt das Feld `cartLimitMode` in `KinematicsConfig`
(Empfänger-Seite), wie der Arm reagiert:

| Modus | Verhalten |
|-------|-----------|
| `CART_LIMIT_CLAMP` | **Standard.** Jedes Gelenk klemmt einzeln an seinem Limit – der Arm fährt so nah wie möglich heran und bleibt am Anschlag stehen. Kann die Pose verzerren. |
| `CART_LIMIT_HOLD` | Sobald ein Gelenk anschlagen würde, hält der Arm die **letzte gültige Pose** statt einer verzerrten Stellung. |
| `CART_LIMIT_PROJECT` | Das Ziel wird auf den **nächstgelegenen erreichbaren Punkt** projiziert: die Reichweite wird begrenzt, Richtung und Werkzeug-Neigung bleiben erhalten. |

```cpp
// Empfänger: in der KinematicsConfig setzen
KinematicsConfig kin = { /* … */ };
kin.cartLimitMode = CART_LIMIT_CLAMP;   // oder _HOLD / _PROJECT
robotLink.setKinematics(kin);
```

## Beispielcode

```cpp
#include <RobotLink.h>

const int xPot       = 4;   // Achse 0 → X (vor/zurück)
const int yPot       = 7;   // Achse 1 → Y (links/rechts)
const int zPot       = 6;   // Achse 2 → Z (oben/unten)
const int pitchPot   = 5;   // Achse 3 → Werkzeug-Neigung η
const int wristPot   = 15;  // Achse 4 → Handgelenk-Rotation
const int gripperPot = 16;  // Achse 5 → Greifer

const int smoothing = 10;   // Glättungsstärke (0 = aus, 50 = stark)

void setup() {
  robotLink.setAxisSmoothing(0, smoothing);
  robotLink.setAxisSmoothing(1, smoothing);
  robotLink.setAxisSmoothing(2, smoothing);
  robotLink.setAxisSmoothing(3, smoothing);
  robotLink.setAxisSmoothing(4, smoothing);
  robotLink.setAxisSmoothing(5, smoothing);

  robotLink.setCoordMode(COORD_CARTESIAN);
  robotLink.beginSender();
}

void loop() {
  robotLink.update();

  // Semantische Setter statt setAxisValue(index, …)
  robotLink.setX(analogRead(xPot));
  robotLink.setY(analogRead(yPot));
  robotLink.setZ(analogRead(zPot));
  robotLink.setPitch(analogRead(pitchPot));
  robotLink.setWristRotate(analogRead(wristPot));
  robotLink.setGripper(analogRead(gripperPot));

  robotLink.sendAllAxes();
  delay(20);  // 50 Hz
}
```

<Aside>
  Der Gruppenkanal (LED-Farbe) ist unabhängig vom Koordinatenmodus. Sender und Empfänger müssen auf demselben Kanal sein – der Kanal wird per Doppel-Reset gewechselt.
</Aside>

## Funktionsweise der 3D-inversen Kinematik

### Schritt 1 – Basiswinkel

Der Basiswinkel ergibt sich aus den X- und Y-Koordinaten plus dem optionalen
[Yaw-Offset](#ausrichtung-drehen-yaw-offset):

```
ψ = atan2(Y, X) + yawOffset
```

### Schritt 2 – Planare Reichweite

Der horizontale Abstand von der Basis zur Werkzeugspitze (minus Handgelenksverlängerung L3):

```
r_eff = sqrt(X² + Y²) - L3 · cos(η)
z_eff = Z - L3 · sin(η)
```

### Schritt 3 – 2-Gelenk-Löser in der Sagittalebene

Identisch zum zylindrischen Modus wird der Cosinus-Satz angewandt, um die Innenwinkel des Dreiecks Schulter–Ellbogen–Handgelenk zu berechnen:

```
d = sqrt(r_eff² + z_eff²)
α = arccos( (L1² + L2² - d²) / (2 · L1 · L2) )   // Innenwinkel am Ellbogen
φ = arccos( (L1² + d²  - L2²) / (2 · L1 · d ) )   // Innenwinkel an der Schulter
β = φ + atan2(z_eff, r_eff)                        // absoluter Schulterwinkel
δ = β + α - π                                      // absoluter Unterarmwinkel
```

### Schritt 4 – Handgelenk-Kompensation

Der Handgelenk-Neigungswinkel kompensiert den Unterarmwinkel, damit das Werkzeug die gewünschte absolute Neigung η beibehält:

```
θ_Handgelenk = η - δ
```

Die Servo-Winkel ergeben sich anschließend aus den mechanischen Null-Offsets: `Schulter = β - π/2`, `Ellbogen = α - π/2`.

<Aside type="caution">
  **Singuläre Positionen:** Direkt über der Basis (X ≈ 0, Y ≈ 0) und am maximalen Ausstreckpunkt kann die IK numerisch instabil werden. In diesen Bereichen können abrupte Bewegungen auftreten. Es empfiehlt sich, diese Positionen im praktischen Betrieb zu vermeiden.
</Aside>

<Aside type="tip">
  Der kartesische Modus reagiert empfindlicher auf schnelle Potentiometer-Bewegungen als der Direkt-Modus, da kleine Wertänderungen große Gelenkwinkeländerungen auslösen können. Höhere Glättungswerte (z. B. 15–20) führen zu ruhigeren Bewegungen.
</Aside>
